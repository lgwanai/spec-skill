# Spec-Skill Canonical Artifact Registry

This directory contains the template files for every artifact that spec-driven development workflows produce. This document is the authoritative index: if a `.planning/` file is not listed here, it is not a canonical artifact and validation tools may flag it.

Agents should query this file before treating a `.planning/` file as authoritative.

---

## `.planning/` Root Artifacts

These files live directly at `.planning/` — not inside phase subdirectories.

| File | Template | Produced by | Purpose |
|------|----------|-------------|---------|
| `PROJECT.md` | `PROJECT.md` | Project init | Project identity, goals, requirements summary |
| `ROADMAP.md` | `ROADMAP.md` | Roadmap planning | Phase plan with milestones and progress tracking |
| `STATE.md` | `STATE.md` | Project init, transitions | Current session state, active phase, last activity |
| `REQUIREMENTS.md` | `requirements.md` | New milestone | Functional requirements with traceability |
| `MILESTONES.md` | `milestone.md` | Milestone completion | Log of completed milestones with accomplishments |
| `BACKLOG.md` | *(inline)* | Backlog capture | Pending ideas and deferred work |
| `LEARNINGS.md` | *(inline)* | Phase execution | Phase retrospective learnings for future plans |
| `config.json` | `config.json` | Project init, health repair | Project-specific workflow configuration |
| `CLAUDE.md` | *(inline)* | Profile setup | Auto-assembled AI context file |

### Version-stamped artifacts (pattern: `vX.Y-*.md`)

| Pattern | Produced by | Purpose |
|---------|-------------|---------|
| `vX.Y-MILESTONE-AUDIT.md` | Milestone audit | Milestone audit report before archiving |

These files are archived to `.planning/milestones/` at milestone completion. Finding them at the `.planning/` root after completion indicates the archive step was skipped.

These files live inside a phase directory.

| File Pattern | Template | Produced by | Purpose |
|-------------|----------|-------------|---------|

| `NN-VERIFICATION.md` | `verification-report.md` | Verification | Post-execution verification report |

## Milestone Archive (`.planning/milestones/`)

Files archived at milestone completion.

| File Pattern | Source |
|-------------|--------|
| `vX.Y-ROADMAP.md` | Snapshot of ROADMAP.md at milestone close |
| `vX.Y-REQUIREMENTS.md` | Snapshot of REQUIREMENTS.md at milestone close |
| `vX.Y-MILESTONE-AUDIT.md` | Moved from `.planning/` root |
| `vX.Y-phases/` | Archived phase directories |

## Adding a New Canonical Artifact

When a new workflow produces a `.planning/` file:

1. Add the file name and purpose to the appropriate table above
2. Add the template to `templates/` if one exists
3. Update the validation tool's canonical file list
4. Document the producing workflow in the relevant workflow doc
